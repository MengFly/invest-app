# API 缓存优化 - 任务规划

## 1. 需求概述

优化基金接口请求的缓存策略，减少不必要的网络请求，提升页面响应速度。缓存默认有效期为半天（12 小时）。

## 2. 需求澄清记录

无需额外澄清，需求明确：已有缓存系统，但 hooks 在缓存有效时仍会发起网络请求。

## 3. 任务列表

### 任务 1：修改缓存 TTL 为半天

| 属性 | 值 |
|------|-----|
| **描述** | 将 `CACHE_TTL` 中三个缓存项的过期时间从 24 小时改为 12 小时（12 * 60 * 60 * 1000） |
| **涉及文件** | `src/services/cache.ts` |
| **验收标准** | 缓存 12 小时后自动过期 |

### 任务 2：缓存有效时跳过网络请求

| 属性 | 值 |
|------|-----|
| **描述** | 修改 `useFundList`、`useFundBasicInfo`、`useFundNetWorth` 三个 hook：当读取到有效缓存时，跳过后续的网络拉取请求。只有缓存不存在或过期时，才发起网络请求 |
| **涉及文件** | `src/hooks/useFund.ts` |
| **验收标准** | 打开页面时，有缓存则不发起网络请求；无缓存或缓存过期时发起请求 |

## 4. 执行顺序

两个任务可以同时开发，互不依赖。
